1. User Journey

End-to-end flow for generating live product demos from URLs.

1

Configure

User enters one or more URLs in the textarea (e.g. product pages, landing pages). Multi-line input supports batch generation.

📝 Configure screen
MyMetaView 5.0 — "Generate live product demos from any URL"
Textarea for URLs, "Generate demo preview" button
2

Submitting

Frontend calls POST /api/v1/demo-v2/batch or POST /api/v1/demo-v2/jobs. Job is queued; UI shows "Creating demo job…"

3

Generating

Backend processes URLs (single or parallel in 6.0). Frontend polls GET /demo-v2/jobs/{id}/status or batch endpoints every ~1.5s. Progress bar and per-URL status update in real time.

⚡ Generating screen
"Generating previews…" — Loading strip animation, demo items in pending/generating state
4

Results

When all URLs complete: success grid with thumbnails, or partial/error states. User can review, download, or export. Live demo storyboard available.

✅ Results screen
"Demo previews ready to review" — Grid of composited preview images with URLs

3. API Architecture (6.0)

6.0 Prefer async jobs for production (avoids 60s timeout). Use batch for multi-URL workloads.

EndpointMethodPurpose
/demo-v2/previewPOSTSync preview (may timeout)
/demo-v2/jobsPOSTCreate async job
/demo-v2/jobs/{id}/statusGETPoll job status
/demo-v2/batchPOSTCreate batch job (up to 50 URLs)
/demo-v2/batch/{id}GETBatch status
/demo-v2/batch/{id}/resultsGETBatch results

4. Batch Parallelization 6.0
